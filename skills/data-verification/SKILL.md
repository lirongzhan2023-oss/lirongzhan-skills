# 数据质量保证技能

## 核心原则

### 1. 结论先行

回答结构：
```
【结论】XXX

【验证过程】
- 方法1：XXX → 结果1
- 方法2：XXX → 结果2

【数据来源】
- 来源1：XXX
- 来源2：XXX
```

### 2. 多次验证规则

**必须验证2遍以上的情况**：
- 任何具体数字（金额、数量、比例）
- 统计结果（销售额、利润率、转化率）
- 排名/对比数据
- 预测/估算数据

**验证方法**：
- 不同计算路径交叉验证
- 倒推验证（用结果推过程）
- 合理性检验（结果在合理区间？）

### 3. 置信度标注

| 标注 | 含义 | 场景 |
|------|------|------|
| ✅ **已验证** | 多重核对，确定准确 | 系统数据、计算复核 |
| ⚠️ **参考** | 单来源或估算 | 行业均值、经验值 |
| ❓ **推测** | 基于有限信息的猜测 | 预测、未证实消息 |

### 4. 必检项

回答前必须检查：
- [ ] 计算过程是否留痕？
- [ ] 结果是否在合理区间？（异常值报警）
- [ ] 数据来源是否注明？
- [ ] 结论是否有充分支撑？

## 错误示例 vs 正确示例

### ❌ 错误
```
今日营业额约1.2万，毛利大概40%左右吧。
```

### ✅ 正确
```
【结论】今日营业额：12,345元 | 毛利率：38.5%

【验证】
- POS系统：12,345元
- 外卖平台后台：12,345元
- 两者一致，✅ 已验证

【毛利计算】
- 收入：12,345元
- 成本：7,592元（食材+包装）
- 毛利率：(12345-7592)/12345 = 38.5%
- 复核一遍，结果一致，✅ 已验证
```

## 触发条件

当用户提问涉及以下内容时，必须启用此技能：
- 任何具体数字、金额、数量
- 统计、汇总、分析
- 排名、对比、占比
- 预测、估算、推算

格式不达标 → 返工重写
数据未验证 → 不给出答案
