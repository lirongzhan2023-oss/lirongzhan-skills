# 多Agent协作技能

## 1. 当前模式

### 单Agent模式
```
用户 → 单一Agent → 回答
```
- 目前默认模式
- 一次只处理一个任务

### 多Agent模式
```
用户 → Agent A（分析）→ Agent B（方案）→ Agent C（审核）→ 汇总回答
```

---

## 2. 如何触发多Agent

### 方式1：明确要求

```
"让数据分析Agent和方案Agent一起讨论一下"
→ Spawn多个子Agent协作
```

### 方式2：复杂任务自动拆分

```
任务：分析门店数据并制定改进方案
→ Agent A：分析数据
→ Agent B：制定方案
→ Agent C：审核优化
→ 汇总输出
```

---

## 3. OpenClaw子Agent用法

### Spawn命令

```bash
# 派生子Agent
openclaw sessions_spawn --task "分析外卖数据" --agentId subagent

# 或在对话中
让数据分析Agent来分析这个表格
```

### 当前限制

| 功能 | 状态 |
|------|------|
| 单Agent | ✅ 正常 |
| 子Agent Spawn | ✅ 支持 |
| 自动讨论 | ❌ 需手动触发 |
| 群聊多Agent | ❌ 需配置 |

---

## 4. 实现讨论的步骤

### 步骤1：Spawn子Agent

```
当需要讨论时：
1. 明确告诉我要讨论
2. 我Spawn多个子Agent
3. 每个子Agent分析角度
4. 汇总讨论结果
```

### 步骤2：手动触发示例

```
【你说】：数据分析Agent和营销Agent讨论一下这个方案

【我做】：
→ Spawn数据分析Agent → 分析数据
→ Spawn营销Agent → 分析方案
→ 汇总两人观点 → 给出综合建议
```

---

## 5. 实际应用

### 场景：制定营销方案

```
1. 数据分析Agent
   → 分析门店数据、竞品数据

2. 营销方案Agent
   → 制定活动策略

3. 财务审核Agent
   → 预算评估、ROI计算

4. 汇总
   → 综合给出最终方案
```

### 操作

```
【你可以说】：
"让数据Agent和方案Agent一起看看这个方案是否可行"

【我做】：
→ 调用2个子Agent
→ 分别分析
→ 汇总结果
```
