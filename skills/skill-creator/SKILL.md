# 技能创建技能

## 1. 技能结构

### 标准目录结构

```
skill-name/
├── SKILL.md              # 必须：技能定义
├── references/           # 可选：参考资料
│   └── template.md
└── scripts/              # 可选：脚本文件
    └── run.js
```

### SKILL.md必须包含

```yaml
---
name: skill-name          # 技能名（英文）
description: |            # 描述（一句话+触发条件）
  技能用途说明
---

# 技能标题

## 1. 功能说明
## 2. 使用场景
## 3. 具体方法
```

## 2. 技能模板

### 基础模板

```yaml
---
name: my-skill
description: |
  简短描述用途。用于：具体场景。
  触发条件：用户提到XXX
---

# 我的技能

## 1. 做什么
## 2. 怎么做
## 3. 示例
```

## 3. 创建流程

### 快速创建

```
1. mkdir skills/新技能名
2. mkdir skills/新技能名/references
3. 写 SKILL.md
4. 测试
```

### 命令

```bash
# 创建目录
mkdir skills/新技能
mkdir skills/新技能/references

# 创建文件
write skills/新技能/SKILL.md "# 内容"
```

## 4. 优化技巧

### 精简原则

| 问题 | 解决 |
|------|------|
| 太长 | 只保留关键信息 |
| 太复杂 | 拆成多个技能 |
| 例子太多 | 留1-2个典型 |

### 结构优化

```
优先：
1. 功能列表（表格）
2. 核心方法（步骤）
3. 示例（代码块）

减少：
1. 原理说明
2. 背景介绍
3. 重复内容
```

## 5. 审核清单

### 必须有

- [ ] name（英文名）
- [ ] description（描述+触发条件）
- [ ] 至少一个功能说明

### 建议有

- [ ] 使用示例
- [ ] 输出格式
- [ ] 注意事项

### 避免

- [ ] 太长（<200行最佳）
- [ ] 术语堆砌
- [ ] 过时内容

## 6. 分类建议

### 按用途分类

| 分类 | 示例 |
|------|------|
| 餐饮 | restaurant-ops, takeout-growth |
| 数据 | excel-reader, data-visualization |

### 命名规范

```
技能名：英文小写+连字符
如：excel-reader, cost-accounting
```

## 7. 实战创建

### 场景：BOSS需要新技能

```
1. 理解需求
   - BOSS想要什么？
   - 解决什么问题？

2. 匹配现有
   - 现有技能能否满足？
   - 需要扩展还是新建？

3. 快速创建
   - 用模板生成
   - 填入核心内容

4. 测试验证
   - BOSS试用
   - 反馈改进
```

### 示例：创建"供应商管理"技能

```yaml
---
name: supplier-management
description: |
  供应商管理专家。用于：(1) 供应商筛选
  (2) 议价技巧 (3) 评估管理
  适用于餐饮采购场景
---

# 供应商管理

## 1. 筛选标准
| 标准 | 说明 |
|------|------|
| 价格 | 同类最低 |
| 质量 | 稳定合格 |
| 配送 | 准时可靠 |

## 2. 议价技巧
1. 多家比价
2. 批量采购
3. 长期合作

## 3. 评估周期
每月评估一次
```
