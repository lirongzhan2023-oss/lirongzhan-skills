# 自动分析技能

## 核心能力

### 1. 数据导入

**支持格式**：
- Excel (.xlsx, .xls)
- CSV (.csv)
- 图片截图（OCR识别）
- 文本数据

**自动识别**：
```
□ 文件类型判断
□ 表格结构识别
□ 列名/表头识别
□ 数据类型判断（数值/文本/日期）
□ 缺失值检测
□ 异常值识别
```

---

### 2. 自动诊断

**分析流程**：

```
Step 1: 数据清洗
- 去除空行/空列
- 统一日期格式
- 处理异常值

Step 2: 指标计算
- 转化率 = 订单/访客×100%
- 客单价 = 收入/单量
- 毛利 = 收入×毛利率
- 其他衍生指标

Step 3: 对标分析
- 与行业基准对比
- 与历史数据对比
- 与目标对比

Step 4: 问题识别
- 红灯指标（严重问题）
- 黄灯指标（一般问题）
- 绿灯指标（正常）
```

---

### 3. 核心指标库

**外卖运营指标**：

| 指标 | 计算方式 | 优秀 | 合格 | 差 |
|------|----------|------|------|-----|
| 转化率 | 订单/访客 | >5% | 3-5% | <3% |
| 复购率 | 回头客/总客 | >25% | 15-25% | <15% |
| 评分 | 实测 | 4.9 | 4.5-4.9 | <4.5 |
| 客单价 | 收入/单量 | >35元 | 25-35元 | <25元 |
| 单量增长率 | (本期-上期)/上期 | >10% | 0-10% | 负增长 |

**问题预警规则**：

| 指标 | 红灯条件 | 诊断原因 |
|------|----------|----------|
| 转化率 | <2% | 主图/价格/菜单问题 |
| 复购率 | <10% | 口味/服务/无会员 |
| 评分 | <4.5 | 差评增加 |
| 单量 | 环比-20% | 竞品/活动/流量 |
| 毛利率 | <30% | 成本失控 |

### 4. 方案自动生成

**问题→方案映射**：

| 问题 | 方案 | 优先级 |
|------|------|--------|
| 转化率低 | 主图优化+菜单调整+满减设计 | 🔴 P0 |
| 复购率低 | 会员体系+私域沉淀 | 🔴 P0 |
| 评分低 | 好评引导+差评处理 | 🟡 P1 |
| 客单价低 | 套餐组合+关联推荐 | 🟡 P1 |
| 单量跌 | 竞品分析+活动策划 | 🔴 P0 |

### 6. 趋势分析

**对比维度**：

| 对比类型 | 说明 | 分析内容 |
|----------|------|----------|
| 环比 | 本周vs上周 | 短期趋势 |
| 同比 | 本月vs上月 | 长期趋势 |
| 跨店 | 店1vs店2 | 门店对比 |
| 跨平台 | 美团vs饿了么 | 渠道对比 |

**趋势输出**：
```
【趋势判断】：📈上升 / 📉下降 / ➡️持平

【原因分析】：
1. 可能原因1
2. 可能原因2

【建议】：
1. 如果上升：乘胜追击
2. 如果下降：及时止损
3. 如果持平：寻求突破
```

---

### 7. 自我进化机制

**每次分析后的进化**：

```
□ 问题识别准确吗？→ 优化问题库
□ 方案有效吗？→ 记录效果
□ 有新问题吗？→ 更新指标库
□ 数据质量如何？→ 改进清洗逻辑
```

**进化触发**：
- 每次数据分析后
- 用户反馈方案效果后
- 发现新指标/新问题后

## 数据格式要求

**Excel表格建议包含**：

| 列名 | 说明 | 示例 |
|------|------|------|
| 日期 | 数据周期 | 2025-03-11 |
| 店铺 | 门店名 | 东郊记忆店 |
| 平台 | 渠道 | 美团/饿了么 |
| 单量 | 订单数 | 78 |
| 收入 | 营业额 | 2213 |
| 访客 | 进店人数 | 2232 |
| 转化率 | 下单比例 | 3.5% |
| 复购率 | 回头客比例 | 16.86% |
| 评分 | 店铺评分 | 4.7 |
| 毛利率 | 毛利比例 | 40% |
