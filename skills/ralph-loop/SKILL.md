# Ralph循环（迭代改进）技能

## 1. 核心循环模型

### Ralph循环 / OODA循环

```
Observe（观察）→ Orient（定位）→ Decide（决策）→ Act（行动）
    ↑                                    ↓
    └────────── 循环迭代 ←──────────────┘
```

| 阶段 | 问题 | 行动 |
|------|------|------|
| Observe | 发生了什么？ | 收集数据 |
| Orient | 意味着什么？ | 分析理解 |
| Decide | 该怎么做？ | 制定方案 |
| Act | 立即执行！ | 行动验证 |

---

## 2. PDCA循环

### 标准版本

```
Plan（计划）→ Do（执行）→ Check（检查）→ Act（处理）
    ↑                                        ↓
    └──────────────── 循环 ──────────────────┘
```

### 餐饮应用

| 阶段 | 餐饮场景 |
|------|----------|
| Plan | 制定月度目标 |
| Do | 执行运营活动 |
| Check | 分析数据结果 |
| Act | 优化调整方案 |

---

## 3. 快速迭代法

### 最小闭环

```
1. 假设 → 快速验证
2. 小范围测试
3. 看数据反馈
4. 快速调整
5. 扩大范围
```

### 时间盒

| 迭代 | 时长 | 产出 |
|------|------|------|
| 日迭代 | 1天 | 快速调整 |
| 周迭代 | 1周 | 方案优化 |
| 月迭代 | 1月 | 战略调整 |

## 4. 复盘方法

### 复盘四步

```
1. 回顾目标
   - 当初目标是什么？
   
2. 评估结果
   - 实际发生了什么？
   - 和目标差距多大？
   
3. 分析原因
   - 为什么会这样？
   - 根本原因是什么？
   
4. 总结经验
   - 下次怎么做？
   - 需要改什么？
```

### 复盘模板

```
【项目】：XXX

【目标】：XXX
【结果】：XXX

【做得好】：
1. XXX

【需改进】：
1. XXX

【下次行动】：
1. XXX
```

## 5. 连续改进

### 改进流程

```
问题 → 根因 → 方案 → 实施 → 验证 → 标准化
                              ↓
                            循环
```

### 餐饮改进案例

```
【问题】：物料成本超标

1. 观察：物料59%，超标4%
2. 定位：采购价偏高
3. 决定：找3家比价
4. 行动：更换供应商
5. 检查：新供应商价格
6. 处理：确认省钱，固定合作
```
