# 客户声音分析技能

## 1. 好评分析

### 好评关键词提取

| 类别 | 常见词 |
|------|--------|
| 口味 | 好吃、美味、芝士多、料足 |
| 服务 | 热情、态度好、耐心 |
| 速度 | 出餐快、配送快 |
| 包装 | 精美、完整、不洒 |
| 性价比 | 便宜、划算、值 |

### 好评模板

```
【好评分析】
关键词：芝士多、出餐快、味道好
占比：30%、25%、20%

→ 优势：口味+速度
→ 继续保持
```

---

## 2. 差评分析

### 差评分类

| 类型 | 原因 | 解决 |
|------|------|------|
| 口味 | 太淡/太咸/不好吃 | 标准化配方 |
| 包装 | 洒了、凉了 | 检查打包 |
| 配送 | 慢、态度差 | 选更好配送 |
| 服务 | 态度差 | 培训 |
| 食材 | 不新鲜 | 采购品控 |

### 差评分析模板

```
【差评分析】
1. 口味问题（40%）
   - 太淡：XX次
   - 太咸：XX次

2. 包装问题（30%）
   - 洒了：XX次
   - 凉了：XX次

3. 配送问题（20%）
   - 慢：XX次

→ 首要改进：口味+包装
```

---

## 3. 竞品对比

### 对比维度

| 维度 | 自己 | 竞品A | 竞品B |
|------|------|-------|-------|
| 评分 | 4.7 | 4.5 | 4.8 |
| 好评关键词 | 出餐快 | 味道好 | 便宜 |
| 差评关键词 | 包装 | 配送 | 服务 |

### 差异化定位

```
【自己优势】：出餐快
【竞品优势】：味道好、便宜

→ 定位：快+料足
→ 宣传：20分钟送达
```

---

## 4. 改进建议

### 问题→方案映射

| 问题 | 方案 |
|------|------|
| 出餐慢 | 预制+高峰期备货 |
| 口味不稳定 | 标准化配方 |
| 包装差 | 换包装供应商 |
| 配送慢 | 优化动线+选配送 |

### 优先级排序

```
问题1：口味（30%）→ 立即改进
问题2：包装（25%）→ 1周改进
问题3：速度（20%）→ 2周改进
```

---

## 5. 评价监控

### 监控指标

| 指标 | 正常 | 预警 |
|------|------|------|
| 评分 | >4.7 | <4.5 |
| 好评率 | >80% | <70% |
| 差评率 | <10% | >15% |
| 差评回复 | <30min | >2h |

### 预警机制

```
每日检查：
□ 评分变化
□ 新增差评
□ 差评回复率

→ 有问题立即处理
```

## 6. 好评引导

### 引导话术

| 场景 | 话术 |
|------|------|
| 用餐结束 | "吃得怎么样？满意给个好评~" |
| 打包时 | "打包回去也欢迎来点评~" |
| 好评后 | "谢谢！欢迎下次再来" |

### 好评激励

| 方式 | 成本 |
|------|------|
| 口头感谢 | 0元 |
| 小礼物 | 1-2元 |
| 优惠券 | 3-5元 |

---

## 7. 差评回复

### 回复模板

| 问题 | 回复 |
|------|------|
| 口味一般 | "抱歉口味没达到预期，我们会继续改进" |
| 配送慢 | "抱歉让您在高峰期等待，已反馈配送优化" |
| 包装差 | "非常抱歉，已安排换新包装" |
| 少送 | "抱歉漏送，请联系客服全额退款" |

### 回复原则

```
1. 30分钟内回复
2. 态度真诚道歉
3. 提出解决方案
4. 私下联系补偿
5. 不删除差评
```

【好评关键词】

【改进建议】
1. 包装改进（紧急）
2. 口味标准化（重要）
3. 出餐速度保持（继续）
