# 文件传输技能

## 1. 常用方法

### 方法1：直接路径（用户可访问时）

```
当文件在用户可访问的路径时：
- 告诉用户文件路径
- 用户自己打开

示例：
"文件在：C:\Users\Administrator\.openclaw\workspace\xxx.xlsx"
```

### 方法2：飞书/微信发送

```
通过飞书消息发送：
- 文件 < 20MB：直接发送
- 文件 > 20MB：需要上传到云盘

OpenClaw飞书渠道可以直接发文件
```

### 方法3：邮件发送

```
发送邮件带附件：
- 附件 < 25MB
- 需要配置SMTP

适合：报告、表格、文档
```

### 方法4：临时链接

```
生成临时下载链接：
- 1-7天有效期
- 需要上传到临时存储

工具：file.io、tmpfiles.com
```

## 2. 当前问题解决

### 场景：用户想要文件

**问题**：文件在本地，用户拿不到

**解决方案**：

| 情况 | 方法 |
|------|------|
| 用户在同一电脑 | 给完整路径 |
| 用户远程 | 需要上传到云/发邮件 |
| 飞书渠道 | 尝试用飞书发送 |

## 4. 应急方案

### 当无法发送文件时

```
1. 告诉用户文件路径
2. 告诉用户内容概要
3. 问用户怎么方便接收

示例：
"文件已做好，包含5个工作表：
1. 供应商询价
2. 领料登记
3. 设备维修
4. 成本分析
5. 执行计划

你想怎么接收？
- 路径打开
- 我尝试飞书发送
- 邮件给你"
```

---

## 5. 最佳实践

```
发送文件前：
□ 确认用户能收到
□ 准备备选方案（路径/邮件）
□ 说明文件内容

发送后：
□ 确认用户是否收到
□ 必要时提供打开方式
```
