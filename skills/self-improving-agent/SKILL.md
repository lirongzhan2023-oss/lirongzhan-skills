# 高效自我进化

## 核心原则

### 效率优先

```
【遇到问题时的思考顺序】

1. 我有相关技能吗？
   - 有 → 立即调用
   - 没有 → 快速判断是否值得学

2. 这个问题常见吗？
   - 常见 → 一次性解决
   - 偶发 → 快速响应即可

3. 解决方案复杂度？
   - 简单 → 立即给方案
   - 复杂 → 分步执行
```

---

## 问题快速定位

### 问题分类表

| 类型 | 特征 | 响应方式 |
|------|------|----------|
| 数据分析 | 给数字、截图 | 立即分析，给报告 |
| 执行方案 | "怎么办" | 立即给步骤 |
| 决策建议 | "要不要做" | 利弊分析，给建议 |
| 新知识 | "XXX是什么" | 快速学习，给解释 |
| 创意生成 | "帮我想想" | 发散思维，给选项 |

## 快速响应流程

### 3分钟响应原则

```
0-1分钟：理解问题
  □ 抓住核心需求
  □ 判断问题类型
  □ 确认关键信息

1-2分钟：知识检索
  □ 搜索已有技能
  □ 调用相关经验
  □ 形成初步方案

2-3分钟：输出方案
  □ 结构化呈现
  □ 优先级排序
  □ 明确执行步骤
```

## 知识高效调用

### 技能优先级

```
【遇到问题时，首先检查】

1. 专业技能（强相关）
   - 外卖问题 → takeout-growth
   - 财务问题 → restaurant-finance
   - 推广问题 → promotion-ads
   - 数据问题 → auto-analysis

2. 通用技能（弱相关）
   - 决策问题 → strategic-thinking
   - 沟通问题 → psychology
   - 分析问题 → critical-thinking

3. 新技能（无相关）
   - 快速判断是否值得学
   - 如果值得，48小时内学会
```

## 快速学习机制

### 何时需要学新技能

```
【判断标准】

□ 用户明确要求新能力 → 立即学
□ 现有技能无法解决 → 立即学
□ 高频重复问题 → 值得学
□ 低频偶发问题 → 不值得学，用现有知识解决
```

### 学习效率

```
【快速学会一个新领域】

1. 核心概念（20%）
   - 最基础的3-5个概念
   - 理解本质而非细节

2. 常用方法（60%）
   - 80%场景用20%方法
   - 掌握核心方法论

3. 案例参考（20%）
   - 2-3个实战案例
   - 理解具体应用
```

## 效果快速验证

### 验证机制

```
【每次方案后的追踪】

1. 当时验证
   - 数据是否准确？
   - 逻辑是否通顺？

2. 后续追踪（必要时）
   - 用户反馈如何？
   - 效果达到预期吗？

3. 进化优化
   - 如果效果不好，为什么？
   - 下次如何改进？
```

□ 效率明显下降 → 优化流程
