# 产品经理技能

## 1. 需求分析

### 用户痛点挖掘

| 方法 | 说明 |
|------|------|
| 用户访谈 | 直接沟通了解需求 |
| 问卷调查 | 大规模收集反馈 |
| 数据分析 | 从行为数据发现痛点 |
| 竞品差评 | 了解对手用户抱怨 |

### 需求优先级

**MoSCoW法则**

| 分类 | 说明 | 占比 |
|------|------|------|
| Must have | 必须有 | 20% |
| Should have | 应该有 | 30% |
| Could have | 可以有 | 40% |
| Won't have | 不会有 | 10% |

**KANO模型**

| 类型 | 特点 |
|------|------|
| 兴奋型 | 有惊喜，无不满 |
| 期望型 | 越多越好 |
| 基本型 | 必须有，少了不满 |
| 无差异型 | 有没有无所谓 |
| 反向型 | 越少越好 |

## 2. 产品设计

### 功能规划

```
【功能清单】
| 功能 | 优先级 | 复杂度 | 负责人 | 状态 |
|------|--------|--------|--------|------|
| 订单管理 | P0 | 中 | A | 进行中 |
| 库存管理 | P1 | 高 | B | 规划中 |

【版本规划】
- V1.0：核心功能
- V1.1：优化功能
- V2.0：新增功能
```

### 流程设计

```
用户流程：
1. 打开APP
2. 选择商品
3. 加入购物车
4. 提交订单
5. 支付成功
```

### 原型概念

```
【页面结构】
- 首页：轮播 + 分类 + 推荐
- 列表页：筛选 + 排序 + 商品
- 详情页：图片 + 价格 + 加入购物车
- 购物车：商品列表 + 数量 + 结算
- 订单页：列表 + 详情 + 售后
```

## 4. 数据分析

### 核心指标

| 指标 | 公式 | 含义 |
|------|------|------|
| DAU | 日活跃用户 | 流量 |
| MAU | 月活跃用户 | 规模 |
| 留存率 | 次日/7日/30日 | 粘性 |
| 转化率 | 付费用户/活跃 | 变现 |
| ARPU | 收入/用户 | 价值 |

### 转化漏斗

```
【电商漏斗】
访问 → 浏览 → 加购 → 下单 → 支付
100% → 60% → 30% → 10% → 8%
```

### A/B测试

```
【测试方案】
- A组：原版按钮
- B组：红色按钮
- 目标：提升点击率
- 结论：红色按钮提升15%
```

## 5. 竞品分析

### 分析框架

| 维度 | 内容 |
|------|------|
| 功能对比 | 对方有什么功能 |
| 用户体验 | 流程是否顺畅 |
| 定价策略 | 价格体系如何 |
| 运营手段 | 推广/活动方式 |
| 优势/劣势 | 总结差异化 |

### 竞品报告模板

```
【竞品A分析】
- 定位：高端市场
- 核心功能：XXX
- 优势：品牌强、体验好
- 劣势：价格高、功能少

【我方策略】
- 定位：性价比
- 差异化：功能丰富、价格低
```
